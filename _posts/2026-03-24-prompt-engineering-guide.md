프롬프트 엔지니어링은 LLM을 효과적으로 활용하기 위한 가장 기본적이면서도 강력한 기술입니다. 이 글에서는 실무에서 바로 적용할 수 있는 핵심 기법들을 소개합니다.

## 1. 역할 부여 (Role Prompting)

모델에게 명확한 역할을 부여하면 답변의 품질이 크게 향상됩니다.

```
당신은 10년 경력의 시니어 백엔드 개발자입니다.
다음 코드를 리뷰하고 개선점을 제안해주세요.
```

## 2. 구조화된 출력 요청

원하는 출력 형식을 명시적으로 지정하세요.

```
다음 형식으로 답변해주세요:
## 요약
(한 문장 요약)

## 핵심 포인트
- 포인트 1
- 포인트 2

## 결론
(실행 가능한 제안)
```

## 3. Chain of Thought (사고의 연쇄)

복잡한 문제는 단계별로 사고하도록 유도합니다.

> "단계별로 생각해보세요" 한 줄만 추가해도 수학, 논리 문제의 정확도가 크게 올라갑니다.

## 4. Few-shot 예시 활용

원하는 결과물의 예시를 2-3개 제공하면 모델이 패턴을 학습합니다.

```
입력: "오늘 날씨가 좋아서 기분이 좋다"
감정: 긍정

입력: "프로젝트 마감일이 다가오는데 진행이 안 된다"
감정: 부정

입력: "새로운 팀원이 왔는데 적응을 잘 할지 모르겠다"
감정:
```

## 5. 제약 조건 명시

답변의 범위와 제약을 명확히 설정하세요.

- 글자 수 제한
- 사용할 언어/프레임워크 지정
- 피해야 할 패턴 명시
- 대상 독자 수준 지정

## 마무리

프롬프트 엔지니어링은 단순한 기술이 아니라 **소통의 기술**입니다. LLM과 효과적으로 소통하는 방법을 익히면, AI를 훨씬 더 강력한 도구로 활용할 수 있습니다.
